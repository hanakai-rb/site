## How it's built

hanakai.org is a [Hanami](/hanami) app. We build Hanakai with Hanakai.

- Written in [Ruby](https://ruby-lang.org) on [Hanami](/hanami).
- Content is authored in Markdown, rendered with [CommonMarker](https://github.com/gjtorikian/commonmarker) and [html-pipeline](https://github.com/gjtorikian/html-pipeline).
- Pages are pre-rendered to static HTML with [Parklife](https://github.com/benpickles/parklife) and deployed to [Netlify](https://www.netlify.com).
- Search is powered by [Pagefind](https://pagefind.app), a fully static, client-side index.
- Open Graph images are generated at build time with [Satori](https://github.com/vercel/satori) and [resvg](https://github.com/RazrFalcon/resvg).
- Styling uses [Tailwind CSS](https://tailwindcss.com).
- A small amount of TypeScript handles progressive enhancement, coordinated using [defo](https://github.com/makenosound/defo). We use [Foresight](https://foresightjs.com) to prefetch pages as you reach for links.

The complete source is on GitHub at [hanakai-rb/site](https://github.com/hanakai-rb/site). Issues, pull requests, and corrections are welcome.

## Typography

- Headings and body text are set in [Faire Sprig Sans](https://faire.world/typefaces/faire-sprig-sans), a friendly humanist sans by Faire. It's warm and a little bit quirky, which suits us well.
- Code and other monospaced text is set in [Maple Mono](https://github.com/subframe7536/maple-font), an open source typeface by Subframe. It's friendly and playful, just like us. Code is fun!

## Privacy

We use [Tinylytics](https://tinylytics.app) for lightweight, privacy-respecting analytics. It does not use cookies, does not track you across sites, and does not collect personal information. We do not run any other third-party trackers.
